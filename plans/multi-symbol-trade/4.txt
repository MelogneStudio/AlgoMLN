## Section A — `src/screens/Settings/SettingsScreen.tsx` — add Index Data card

Add a new card section after the existing "Default Backtest Capital" card and before the "About" card. The card must use the **exact same card structure** as existing Settings cards — no new layout primitives.

The card content:

1. **Header**: "Index Data" title + subtitle "22 NSE indices · auto-updated quarterly".
2. **Status grid**: For each of the 22 indices (loaded via `listIndices()` IPC on mount, same pattern as `listStrategies` on the Strategies screen), show:
   - Index display name
   - Symbol count (or "–" if 0)
   - Last updated date (or "never")
3. **Refresh button**: `Button` component with `variant="ghost"`. Label: "Refresh Now". Disabled while `isRefreshing`. On click: call `refreshIndices()`, await result, update state. Show a brief inline success/error message below the button.

State:
```tsx
const [indices, setIndices] = useState<IndexInfo[]>([]);
const [isRefreshing, setIsRefreshing] = useState(false);
const [refreshMsg, setRefreshMsg] = useState<string | null>(null);

useEffect(() => {
  listIndices().then(setIndices);
}, []);

const handleRefresh = async () => {
  setIsRefreshing(true);
  setRefreshMsg(null);
  try {
    const result = await refreshIndices();
    const ok = result.refreshed.length;
    const fail = result.failed.length;
    setRefreshMsg(
      fail === 0
        ? `All ${ok} indices refreshed.`
        : `${ok} refreshed, ${fail} failed.`
    );
    // Reload the updated index list
    listIndices().then(setIndices);
  } catch (e) {
    setRefreshMsg('Refresh failed — check network connection.');
  } finally {
    setIsRefreshing(false);
  }
};
```

The status grid should be scrollable if 22 rows overflow the card's allocated height. Use `overflow-y: auto` with a `max-height` matching the existing card body style in the Settings screen.
